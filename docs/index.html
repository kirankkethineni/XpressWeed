Abstract

The core idea in one paragraph

Weeds compete with primary crops for resources, causing significant yield loss. While CNNs have been applied to weed segmentation, building a single large-scale model is computationally infeasible, and conventional few-shot methods fail when limited images cannot capture the extreme variability in leaf shape, orientation, and illumination. XpressWeed addresses this by modeling plant leaves as textures rather than objects. A Fully Convolutional Network is pre-trained on 12 diverse leaf textures to learn invariant, species-agnostic representations, then a MAML strategy adapts this model to 4 unseen weed species using only 80 images — achieving 85% accuracy versus 48% for traditional MAML.

The Problem

Why standard deep learning fails for weed segmentation

🌿

Shape Variability

Same species looks completely different when twisted, overlapping, or partially occluded. Object-centric models trained on shapes fail.

☀️

Lighting Changes

Color hues shift dramatically under field lighting. RGB and color-based approaches are unreliable in real farmland conditions.

🔬

New Weed Species

Weeds evolve regionally across seasons. A model trained in one region fails in another — retraining from scratch is impractical.

📷

Data Scarcity

Pixel-level annotation of weed images is expensive. A few dozen images is all that's available for new species in practice.

The Key Insight

Leaves are textures, not rigid objects

💡

Texture-Based Representation

Each plant species exhibits distinctive texture patterns — vein distributions, surface roughness, micro-structures — that remain relatively invariant to shape deformations and occlusions. Unlike object-centric representations that rely on global shape, texture representations emphasize local spatial statistics, which are stable under the variations pervasive in real farmland imagery.

By pre-training a CNN segmentation model on diverse leaf textures, XpressWeed learns filters that generalize across species. These texture priors then provide a powerful initialization for few-shot adaptation with MAML.

Stage 1 — Supervised Pre-training

  • Dataset: PlantVillage (12 healthy species)
  • Images arranged into collages with overlapping leaves
  • Simulates real farmland conditions (tangled grasses)
  • Loss: Weighted Cross-Entropy (handles class imbalance)
  • Result: 96% train acc · 92% val acc · 92% mIoU

Stage 2 — MAML Adaptation

  • Dataset: Real Indian farmland (4 unseen weed species)
  • 80 patches cropped from 20 high-res (2112×1600) images
  • Episodes: 3-way · 12 support + 10 query per class
  • 25 epochs · Inner loop → support · Outer loop → query
  • N-way episodic training = implicit contrastive learning

Network Architecture

U-Net-inspired encoder-decoder optimized for texture discrimination

🔽

Encoder

Progressive downsampling via stacked conv blocks. 512×768 → bottleneck 8×12×512. Captures high-level texture features stable under deformation.

🔗

Skip Connections

Intermediate feature maps saved at each scale. Decoder concatenates encoder maps for multi-scale fusion of global texture + fine-grained edges.

🔼

Decoder

Feature-fusion upsampling (not transposed conv). Boundary refinement is implicit — no expensive deconvolution operations.

📦

Lightweight by Design

Separable convolutions reduce parameters. MAML requires second-order gradients — smaller models converge faster during adaptation.

MAML Episode Construction

# Each training episode:
#   - 3-way classification (3 classes per episode)
#   - 12 support samples per class  (inner loop update)
#   - 10 query  samples per class   (outer loop meta-update)

for epoch in range(25):
    for episode in episodes:
        support_x, support_y = episode['support']   # (36, 512, 768, 3)
        query_x,   query_y   = episode['query']     # (30, 512, 768, 3)

        # Inner loop: fast adapt on support set
        fast_weights = inner_loop_update(model, support_x, support_y)

        # Outer loop: meta-update on query set
        meta_loss = compute_loss(fast_weights, query_x, query_y)
        meta_update(model, meta_loss)
